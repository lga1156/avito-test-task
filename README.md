### **О проекте**

---

Личный кабинет продавца с интегрированным AI-ассистентом, который помогает улучшать описания объявлений. Продавец видит список своих объявлений, выбирает нужное и может просмотреть карточку товара, перейти к редактированию и запросить рекомендации от нейросети. Сервис анализирует содержимое карточки товара и предлагает улучшения текста.

### **Технологический стек**

---

- Node.js v25.1
- React v19
- `react-router-dom` для роутинга
- TypeScript v5.9
- UI-библиотека Mantine v8
- Стейт-менеджмент — Zustand v5
- Система сборки — Vite 8
- HTTP-запросы — связка React Query + Axios

---

Приложение состоит из 4 страниц (3 уникальных):

---

<img width="1470" height="956" alt="ads-list" src="https://github.com/user-attachments/assets/9b75e175-07f9-4cb7-a56b-0efd7af7d467" />

Шапка страницы:

- Заголовок «Мои объявления» с общим количеством объявлений
- Строка поиска по названию объявления
- Сортировка (по новизне, по цене, по названию).
 
Каждая карточка содержит:
 
- Изображение (placeholder)
- Категорию («Электроника», «Транспорт», «Недвижимость» и т. д. )
- Название объявления
- Цену
- Бейдж «Требует доработок» (если у объявления есть незаполненные поля).
 
Фильтры (боковая панель):
 
- Фильтрация по категории: «Электроника», «Транспорт», «Недвижимость» и т. д.
- Переключатель «Только требующие доработок»
- Кнопка «Сбросить фильтры».
 
Пагинация:
 
- По 10 объявлений на страницу
- Навигация между страницами (с кнопками «Вперёд» / «Назад»).
 
Действия:
 
- При клике на карточку — переход на страницу просмотра объявления.
- Переключатель layout: сетка/список.

---

Отображаемая информация:

### **3. Страница редактирования объявления (/ads/:id/edit)**

---

<img width="1470" height="956" alt="edit" src="https://github.com/user-attachments/assets/882f280d-afce-4065-899b-7c9f540672cb" />

Форма редактирования существующего объявления.
 
Поля формы:
 
- Категория
- Название
- Цена
- Характеристики — набор полей зависит от выбранной категории
- Описание
 
AI-функции:

![ai-agent](https://github.com/user-attachments/assets/6add2475-b672-42db-bdc1-6f3bbf9ca5e3)
 
- Кнопка «Придумать описание» / «Улучшить описание» — отправляет данные объявления в LLM и предлагает сгенерированное описание, которое можно автоматически подставить в поле по кнопке «Применить»
- Кнопка «Узнать рыночную цену» — отправляет данные объявления в LLM и предлагает актуальную цену.


 
Действия:
 
- **Кнопка «Сохранить»** сохраняет изменения через PUT/PATCH-запрос к API, после успешного изменения — редирект на страницу объявления
- **Кнопка «Отменить»** возвращает к странице просмотра объявления без сохранения.
 
Сохранение черновика:
 
- Изменения сохраняются в localStorage, данные можно восстановить при случайном обновлении страницы.

## Как запустить

---

Для удобства проверки весь проект (Frontend + Backend) обернут в Docker. Никаких дополнительных установок локально не требуется.

### 1. Клонирование репозитория

---

```bash
git clone https://github.com/lga1156/avito-test-task.git
cd avito-test-task
```

### 2. Настройка переменных окружения

---

В проекте реализована генерация описаний и оценка стоимости через Google Gemini API.
Секретные ключи не коммитятся в репозиторий по соображениям безопасности. Вам нужно создать свой файл конфигурации на основе предоставленного шаблона:

```
# Копируем шаблон
cp .env.example .env
```

Далее откройте созданный файл `.env` и вставьте ваш персональный API-ключ:

```
VITE_GEMINI_API_KEY=Тут_ваш_ключ...
```

### 3. Сборка и запуск контейнеров

---

Запустите Docker Compose. Он самостоятельно соберет фронтенд, поднимет backend-сервер и раздаст статику через Nginx.

```
docker-compose up --build -d
```

Готово! Можете переходить в браузер по адресу `http://localhost:3000`

## Что реализовал

---

- layout: сетка/список
- AI-ассистент для улучшения объявления
- Прерывание (отмена) запросов при переходе между страницами
- Возможность запустить через `Docker`
- Для корректной работы с Backend внес некоторые изменения в файлы директории `/server` (правки пометил комментариями)
- Использован Zustand для легковесного хранения локального UI-состояния (сетка/список) и параметров поиска.
